# Windows環境でのlocalhost/index.php表示遅延の診断手順

## 問題概要
Windows機でChromeでlocalhostのindex.phpを表示する際に、約1分程度の時間がかかる問題の原因を特定するための診断手順です。

---

## 診断ツールの使用（推奨：最初に実行）

### 自動診断スクリプト

#### PowerShellスクリプト（基本的な確認）
```powershell
# PowerShellで実行
cd "D:\AI関連\cursor\pocketnavi_08_new"
.\scripts\diagnose_performance.ps1
```

このスクリプトは以下を確認します：
- localhostの名前解決
- hostsファイルの設定
- ログファイルの確認
- PHPの確認
- ディスクI/O
- ネットワーク接続
- 関連プロセスの確認

#### PHPスクリプト（詳細な確認）
```powershell
# PowerShellで実行
cd "D:\AI関連\cursor\pocketnavi_08_new"
php scripts\diagnose_performance.php
```

このスクリプトは以下を確認します：
- PHP設定
- データベース接続とクエリ実行時間
- ファイルシステムの読み込み時間
- 環境変数の設定
- キャッシュディレクトリの状態
- 外部リソースへのアクセス時間
- メモリ使用量

**推奨順序：**
1. まずPowerShellスクリプトを実行して基本的な問題を確認
2. 次にPHPスクリプトを実行して詳細な情報を取得
3. 結果に基づいて、以下の手動診断手順を実施

---

## 診断手順

### フェーズ1: 基本的な確認（5-10分）

#### 1.1 ブラウザ開発者ツールでの確認
1. Chromeで `F12` を押して開発者ツールを開く
2. **Network**タブを開く
3. ページをリロード（`Ctrl+R`）
4. 以下を確認：
   - **どのリソースが遅延しているか**（赤いバーや長い時間がかかっているリクエスト）
   - **Waterfall**（各リソースの読み込み順序と時間）
   - **Total Time**（合計時間）
   - **TTFB (Time To First Byte)**（サーバー応答時間）

#### 1.2 PHPエラーログの確認
```powershell
# PowerShellで実行
Get-Content "D:\AI関連\cursor\pocketnavi_08_new\logs\php_errors_*.log" -Tail 50
Get-Content "D:\AI関連\cursor\pocketnavi_08_new\logs\application_*.log" -Tail 50
```

#### 1.3 サーバー側の実行時間測定
- `index.php`の1735-1753行目のパフォーマンス測定コードを有効化
- ブラウザでアクセス後、エラーログを確認して各処理の時間を確認

---

### フェーズ2: ネットワーク関連の確認（10-15分）

#### 2.1 localhostの名前解決確認
```powershell
# PowerShellで実行
ping localhost
nslookup localhost
```

**確認ポイント：**
- `localhost`が`127.0.0.1`に正しく解決されているか
- 名前解決に時間がかかっていないか（1秒以上は異常）

#### 2.2 hostsファイルの確認
```powershell
# PowerShellで実行（管理者権限が必要な場合あり）
notepad C:\Windows\System32\drivers\etc\hosts
```

**確認ポイント：**
- `localhost`のエントリが正しく設定されているか
- 不要なエントリや競合がないか

#### 2.3 ファイアウォール・セキュリティソフトの確認
- Windows Defenderやサードパーティのセキュリティソフトがlocalhost通信をブロックしていないか
- 一時的に無効化してテスト（テスト後は必ず再有効化）

---

### フェーズ3: PHP・サーバー設定の確認（10-15分）

#### 3.1 PHP設定の確認
```powershell
# PHPの設定を確認
php -i | Select-String "max_execution_time"
php -i | Select-String "memory_limit"
php -i | Select-String "default_socket_timeout"
```

#### 3.2 データベース接続の確認
```powershell
# データベース接続テスト
php check_local_setup.php
```

**確認ポイント：**
- データベース接続に時間がかかっていないか
- 接続タイムアウトが発生していないか

#### 3.3 外部リソースの読み込み確認
`index.php`で読み込んでいる外部リソース：
- Google Analytics (`gtag.js`)
- Bootstrap CDN
- Leaflet CDN
- Lucide Icons

**確認方法：**
- 開発者ツールのNetworkタブで、これらのリソースの読み込み時間を確認
- オフライン環境でテスト（外部リソースをコメントアウト）

---

### フェーズ4: 詳細なパフォーマンス測定（15-20分）

#### 4.1 パフォーマンス測定コードの有効化
`index.php`の1735-1753行目を有効化：

```php
register_shutdown_function(function() {
    global $_perf;
    $total = round((microtime(true) - $_perf['start']) * 1000, 2);
    
    error_log("━━━━━━━━━━━━━━━━━━━━━━━━━━");
    error_log("⏱️  TIMING BREAKDOWN");
    error_log("━━━━━━━━━━━━━━━━━━━━━━━━━━");
    
    $prev = 0;
    foreach ($_perf['points'] as $point) {
        $delta = $point['elapsed'] - $prev;
        error_log(sprintf("[%6.2fms] (+%6.2fms) %s", $point['elapsed'], $delta, $point['label']));
        $prev = $point['elapsed'];
    }
    
    error_log("━━━━━━━━━━━━━━━━━━━━━━━━━━");
    error_log("Total: {$total}ms");
    error_log("━━━━━━━━━━━━━━━━━━━━━━━━━━");
});
```

#### 4.2 各処理ポイントでの時間測定を追加
`index.php`の主要な処理ポイントに`markTime()`を追加：

```php
markTime('After Security Headers');
markTime('After Env Load');
markTime('After DB Connection');
markTime('After Functions Load');
markTime('After App Init');
markTime('After Search');
markTime('After View Render');
```

#### 4.3 ログの確認
```powershell
# リアルタイムでログを監視
Get-Content "D:\AI関連\cursor\pocketnavi_08_new\logs\php_errors_*.log" -Wait -Tail 0
```

### フェーズ5: データベース関連の確認（10-15分）

#### 5.1 データベースクエリの実行時間確認
- `src/Services/CachedBuildingService.php`のクエリ実行時間をログ出力
- スロークエリログを有効化（MySQL/MariaDBの場合）

#### 5.2 キャッシュの確認
- キャッシュが正しく機能しているか
- キャッシュファイルの読み込みに時間がかかっていないか

#### 5.3 データベース接続プールの確認
- 接続が適切に再利用されているか
- 接続の確立に時間がかかっていないか

---

### フェーズ6: ブラウザ・拡張機能の確認（5-10分）

#### 6.1 シークレットモードでのテスト
- Chromeのシークレットモード（`Ctrl+Shift+N`）でアクセス
- 拡張機能の影響を除外

#### 6.2 拡張機能の無効化
- すべての拡張機能を一時的に無効化してテスト
- 特に開発者ツール系の拡張機能に注意

#### 6.3 キャッシュのクリア
- `Ctrl+Shift+Delete`でキャッシュをクリア
- ハードリロード（`Ctrl+F5`）でテスト

---

### フェーズ7: ファイルシステム関連の確認（10-15分）

#### 7.1 ファイル読み込み時間の確認
- `require_once`で読み込んでいるファイルの読み込み時間を測定
- 特に大量のファイルを読み込んでいる箇所を確認

#### 7.2 ディスクI/Oの確認
```powershell
# PowerShellでディスク使用率を確認
Get-Counter "\PhysicalDisk(*)\% Disk Time"
```

#### 7.3 ウイルススキャンの影響確認
- リアルタイムスキャンがファイルアクセスを遅延させていないか
- プロジェクトフォルダをスキャン除外リストに追加してテスト

## よくある原因と対処法

### 1. localhostの名前解決遅延
**症状：** TTFBが非常に長い（30秒以上）
**対処：** `hosts`ファイルに`127.0.0.1 localhost`を明示的に追加

### 2. データベース接続タイムアウト
**症状：** データベース接続処理で長時間待機
**対処：** 接続タイムアウト設定を確認・調整

### 3. 外部リソースの読み込み遅延
**症状：** Google AnalyticsやCDNリソースの読み込みに時間がかかる
**対処：** 外部リソースを非同期読み込みに変更、またはローカルにホスト

### 4. キャッシュ処理の遅延
**症状：** キャッシュファイルの読み込み・書き込みに時間がかかる
**対処：** キャッシュディレクトリのパーミッション、ディスクI/Oを確認

### 5. セキュリティソフトの影響
**症状：** ファイルアクセスやネットワーク通信が遅延
**対処：** プロジェクトフォルダを除外リストに追加

### 6. PHPの実行時間制限
**症状：** スクリプトがタイムアウトしている
**対処：** `max_execution_time`を確認・調整

## 緊急時の簡易診断（5分）

時間がない場合の最小限の確認：

1. **Chrome開発者ツールのNetworkタブ**で遅延しているリソースを特定
2. **エラーログ**を確認してエラーや警告がないか確認
3. **シークレットモード**でアクセスして拡張機能の影響を除外
4. **`php check_local_setup.php`**でデータベース接続を確認

これらの結果から、問題の大まかな方向性を特定できます。

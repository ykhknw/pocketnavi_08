# キャッシュファイル自動クリーンアップ

## 概要

`cleanup_cache_files.php` は、検索キャッシュファイルを自動的にクリーンアップするCRONスクリプトです。

## 機能

1. **ファイル数制限**: 指定した最大ファイル数を超えた場合、古いファイルから削除
2. **保持期間制限**: 指定した日数を超える古いファイルを自動削除
3. **バッチ処理**: サーバー負荷を抑えながら段階的に削除
4. **詳細ログ**: すべての処理をログファイルに記録

## 使用方法

### 基本的な実行

```bash
/usr/local/php/8.3/bin/php /path/to/cleanup_cache_files.php
```

### 引数付き実行

```bash
/usr/local/php/8.3/bin/php /path/to/cleanup_cache_files.php [最大ファイル数] [最大保持日数]
```

**例:**
- ファイル数を30,000件、保持期間を3日に制限
```bash
/usr/local/php/8.3/bin/php /path/to/cleanup_cache_files.php 30000 3
```

## CRON設定例

### 推奨設定（毎日午前4時に実行）

```cron
0 4 * * * /usr/local/php/8.3/bin/php /home/username/public_html/scripts/cleanup_cache_files.php >> /home/username/logs/cache_cleanup.log 2>&1
```

### その他の設定例

#### 1. 毎日午前3時（デフォルト設定: 50,000件、7日）
```cron
0 3 * * * /usr/local/php/8.3/bin/php /path/to/cleanup_cache_files.php
```

#### 2. 12時間ごと（ファイル数: 40,000件、保持: 5日）
```cron
0 */12 * * * /usr/local/php/8.3/bin/php /path/to/cleanup_cache_files.php 40000 5
```

#### 3. 週1回日曜日の午前2時（ファイル数: 60,000件、保持: 14日）
```cron
0 2 * * 0 /usr/local/php/8.3/bin/php /path/to/cleanup_cache_files.php 60000 14
```

#### 4. 6時間ごと（高トラフィックサイト向け: 30,000件、3日）
```cron
0 */6 * * * /usr/local/php/8.3/bin/php /path/to/cleanup_cache_files.php 30000 3
```

## デフォルト設定

- **最大ファイル数**: 50,000件
- **最大保持日数**: 7日
- **バッチサイズ**: 1,000件ずつ処理
- **ログファイル**: `../logs/cleanup_cache_files_cron.log`

## 削除の優先順位

1. **保持期間超過ファイル**: 設定日数を超える古いファイルを優先削除
2. **ファイル数超過分**: 上記に加えて、ファイル数が上限を超える場合は古い順に削除

## パフォーマンス

- 50,000件のファイルスキャン: 約1-2秒
- 1,000件の削除: 約0.5-1秒
- メモリ使用量: 通常5-10 MB

## トラブルシューティング

### ディレクトリが見つからない

```
❌ キャッシュディレクトリが見つかりません
```

**対処法**: スクリプト内の `$possiblePaths` にキャッシュディレクトリのパスを追加

### 権限エラー

**対処法**: スクリプトとログディレクトリに適切な権限を設定
```bash
chmod 755 /path/to/cleanup_cache_files.php
chmod 775 /path/to/logs/
```

## 既存スクリプトとの連携

### cleanup_search_history.php との違い

| 項目 | cleanup_cache_files.php | cleanup_search_history.php |
|------|-------------------------|----------------------------|
| 対象 | キャッシュファイル | データベースレコード |
| 場所 | ../cache/search/*.cache | global_search_history |
| 削除基準 | ファイル数・日数 | 日数のみ |
| 推奨実行頻度 | 毎日 | 毎週 |

両方を併用することで、ファイルとデータベースの両方を最適な状態に保つことができます。

## 推奨CRON設定（2つのスクリプト併用）

```cron
# キャッシュファイルクリーンアップ（毎日午前4時）
0 4 * * * /usr/local/php/8.3/bin/php /path/to/cleanup_cache_files.php

# 検索履歴クリーンアップ（毎週日曜午前3時51分）
51 3 * * 0 /usr/local/php/8.3/bin/php /path/to/cleanup_search_history.php
```
